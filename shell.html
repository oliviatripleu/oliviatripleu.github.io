# Shell
- interprets commands
- bash
- most popular shell
### Hash Bang (`#!`)
- start the shell script with a **shebang** line:
- `#!/bin/sh` - tells the operating system to get that program and use it to run the commands in this file
- saying it’s an interpreted language (in this case, shell)
### POSIX-compliance shell:
- dash: `#!/bin/dash`
## Shell script
- `ls` has lots of options:
- e.g. `ls -a -l -S`
- tedious to enter all the options every time
- write a **shell script** so that when you just run `myls` it will run all those options of `ls`
- `vi myls`: ```bash #!/bin/dash ls -laS "$@" ```
- can drop it in a file, but then you can only run `./myls` if you’re in the same directory, or if you give it the path
- not convenient
- when you run `which date`
- it will give you the directory that the `date` command is in
- in shell scripts, you should put the long form for **options**
- for readability
- e.g. `--dereference-command-line` instead of `-H`
- use `#` for comments
### $PATH
- when you run `echo $PATH`
- it will give you a list of directories that it could be in
- so when you run the command `date` it will check in each of the directories in the path, and if it’s any of them, it will run it for you
### tr
- can be used to change spaces into newlines
- `tr ' ' '\n' - change uppercase into lowercase
- `tr 'A-Z' 'a-z' ### chmod
- `chmod u+x frequency.sh` gives you permission to execute the file
- `rwxr--r-x`
## Single vs. Double Quotes
- `' '` groups the characters into a single word
- `" "` expands variables, commands, and arithmetic
## Exit Status and Control - shell scripts look at other programs’ exit status
- false = 1, true = 0
- `echo $?` gives you the exit status of the command you just ran
### Using Exit Status for Conditional Execution
- commands separated by a `;` or newline
- all commands are executed
- `cmd1; cmd2; cmd3` - commands separated by `&&`
- execution stops if command has **non-zero** exit status
- `cmd1 && cmd2 && cmd3`
- commands separated by `||` - execution stops if a command has **zero** exit status
- `cmd1 || cmd2 || cmd3`
- **`exit 1` (break)**
- if the first cmd doesn’t exit with `0`, break
## Test `[]` and Arithmetic `$(())`
- `test -e` can be used to check if a file exists in the current directory - `-eq` equals
- `-lt` less than
- `!` not
- `$#` is number of command line arguments
- `$0` is program name
- `$a` is all the arguments - `1>&2` makes the error message go to `stderr` instead of `stdout`
### Curl
- `curl` gives you the contents of a webpage
- e.g. `curl http://0.0.0.0:8000/` gives you `hello Andrew`
- `--silent` prevents it giving you a bunch of timing information
- e.g. `curl --silent https... 2>/dev/null | grep "$regex"`
## Temporary files - if you change a variable inside a pipeline, it won’t be changed outside the pipeline - `mktemp` to make a temporary file - use `trap 'rm -f $temporary_file' EXIT` - right after you make the temp file - to remove the temporary file when you exit, however the program exits ## Using here document - `<< marker` calls a here document - `marker` ends the here document
